# Contributing to TECHANA

We welcome contributions from the community! By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## How to Contribute

### Reporting Bugs

1. **Search existing issues** — Check if the bug has already been reported
2. **Create a new issue** — Include:
   - Clear, descriptive title
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots or logs (if applicable)
   - Environment information (OS, version, dependencies)

### Suggesting Enhancements

1. **Open an issue** — Describe the feature and its use case
2. **Discuss** — Engage with maintainers and community to refine the proposal
3. **Await approval** — Features may require RFC (Request for Comments) for significant changes

### Pull Requests

1. **Fork the repository** and create a feature branch from `main`
2. **Follow naming conventions**: `feat/`, `fix/`, `docs/`, `refactor/`, `chore/`
3. **Write tests** — All new features and bug fixes must include tests
4. **Pass CI** — Ensure all tests pass and linting is clean
5. **Update documentation** — Update README.md, docs, or inline comments as needed
6. **Squash commits** — Keep commit history clean and logical

## Code Style

- Follow existing patterns and conventions in the codebase
- Use descriptive variable and function names
- Write small, focused commits
- Include clear commit messages
- Prefer simplicity and readability over cleverness

## Review Process

1. **Automated checks** — CI must pass (tests, linting, build)
2. **Maintainer review** — At least one maintainer must approve
3. **Address feedback** — All review comments must be resolved
4. **Merge** — Maintainers will merge approved PRs

## Recognition

All contributors will be recognized in the project's contributors list. Significant contributions may receive additional recognition.
